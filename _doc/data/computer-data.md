# Computer Data
: 컴퓨터 시스템에 의해 처리, 저장, 전송되는 모든 형태의 데이터를 의미  
: [컴퓨터 유형](../computer/computer-type.md), [운영체제](../os/index.md) 등 다양한 요소에 따라 데이터 표현이나 처리 방식에 영향을 미침  

> 컴퓨터 > 운영체제 > 파일 시스템 > 파일 > 데이터


**데이터 표현 방식**

컴퓨터 | 데이터 | 특징
---|---|---
Analog Computer  | 아날로그 데이터 | 온도, 전압, 파장, 소리 등 물리적 데이터
Digital Computer | 디지털 데이터 | 모든 데이터를 0과 1의 이진수로 표현
Quantum Computer | 양자 데이터 | 양자 비트를 통해 여러 상태를 동시에 표현


**데이터 처리 방식**  
: 운영체제는 [파일 시스템](./data-storage/file-system.md)을 통해 데이터를 물리적으로 저장하고 관리함  
: 운영체제마다 파일 시스템이 다르므로 데이터 처리 방식이 달라짐  

운영체제 | 파일 시스템 | 실행 파일 
---|---|---
Windows    | NTFS, FAT32, exFAT | exe, dll, msi 
Linux/Unix | ext4, XFS, Btrfs | sh, deb, rpm


**파일**  
: 데이터를 논리적으로 저장하는 단위  
: 파일 시스템은 데이터를 블록으로 나누어 저장하며 파일은 이러한 블록들의 집합  


**데이터 범위**  
- 사용자 데이터 : 문서, 이미지, 오디오, 비디오 등
- 프로그램 데이터 : 소스 코드, 실행 파일, 라이브러리 등
- 메타데이터 : 파일 속성, 생성 날짜, 수정 날짜 등
- 시스템 데이터 : 운영체제 파일, 설정 정보 등



## Digital Data
: 디지털 컴퓨터가 처리 가능한 데이터로 이진수 조합을 통해 표현됨  
: 텍스트, 이미지, 비디오 등 모든 데이터는 이진 형태로 변환되어 저장되고 처리됨  
: 특정 목적에 따라 다른 진법으로 변환하여 사용될 수 있지만 최종적으로는 이진법으로 처리됨  

1. 입력된 데이터를 [이진 데이터](#binary-data)로 변환해 기억장치나 저장장치에 저장       
2. 필요에 따라 저장된 데이터를 가공(처리)하고
3. 가공된 데이터는 사람이 알아 볼 수 있는 형식으로 변환해 출력장치를 통해 출력

=> 데이터는 입력과 출력시 변환이 필요함



### Binary Data
: 이진 데이터란 0과 1로 구성된 이진 형식으로 변환된 데이터를 의미  
: 데이터는 [인코딩](./data-conversion/encoding/index.md)을 통해 이진 데이터로 변환되며 용량이 큰 데이터는 [압축](./data-conversion/data-compression.md)을 통해 크기를 줄여 저장됨  

```
텍스트 데이터 => 인코딩 => 이진 데이터
이미지 데이터 => 인코딩 => 이진 데이터 => 압축 => 압축된 이진 데이터
```


**Binary Number**  
: 이진수란 0과 1 두 개의 숫자를 사용해 수를 표현하는 방식  

```
이진수 = 1010
십진수 = 1*2³ + 0*2² + 1*2¹ + 0*2 => 8 + 0 + 2 + 0 = 10
```


**Binary System**  
: 2진법이란 한 자리에 0과 1 숫자만을 사용하고 값이 1을 초과하면 자리 올림하여 수를 표현하는 방법  


**Number System**  
: 기수에 따라 다양한 진법이 존재하며 컴퓨터에서는 주로 2진법, 8진법, 10진법, 16진법을 사용  

진법 | 범위 | 표기법(접두사) | 사용
:---:|:---:|:---:|---
2진법  | 01               | 0b | 논리회로, 기계어
8진법  | 01234567         | 0  | 레거시 프로그램
10진법 | 0123456789       | -  | 사용자 인터페이스
16진법 | 0123456789abcdef | 0x | 메모리 주소, 색상 코드, 바이트 단위 데이터


진법의 각 숫자는 숫자가 위치한 자리의 값을 갖음
```
1111011 = 1*2**6 + 1*2**5 + 1*2**4 + 1*2**3 + 0*2**2 + 1*2**1 + 1*2**0
123 = 1*10**2 + 2*10**1 + 3*10**0 
```


**Binary Format**  
: 이진 형식이란 데이터를 이진수로 표현하는 형식을 의미  
: 초기 컴퓨터는 천공 카드의 구멍 유무로 이진 형식을 표현  
: 현대 컴퓨터는 전기 회로에서 전류가 흐르는 상태 유무로 이진 형식을 표현  


**Binary Stream**  
: 이진 데이터의 연속적인 흐름을 의미  
: 파일 입출력, 네트워크 전송 등 데이터 이동이나 처리에 사용  



### Digital Data Unit
: 디지털 데이터의 단위는 물리적 단위와 논리적 단위로 구분됨  
: 물리적 단위는 저장장치에 데이터를 실제로 저장하거나 네트워크에서 데이터를 전송하는데 사용  
: 논리적 단위는 사용자가 데이터를 접근하고 관리하기 위한 방식으로 사용  

- [Physical Unit](#physical-unit)
- [Logical Unit](#logical-unit)



#### Physical Unit
: 컴퓨터 시스템의 물리적 장치에서 데이터를 저장하고 처리하는데 사용되는 단위  
: 최소 단위는 비트이며 최근에는 장치의 용량 증가로 바이트 단위를 기본으로 사용함  

단위 | 변환 
:---:|:---:
bit      | 1 bit
quarter  | 2 bits
nibble   | 4 bits
Byte     | 8 bits
Word     | 16 bits
Kilobyte | 1024 bytes
Megabyte | 1024 kilobytes
Gigabyte | 1024 megabytes
Terabyte | 1024 gigabytes
Petabyte | 1024 terabytes


**Bit**  
: Binary Digit  
: 0 또는 1의 상태를 갖는 가장 작은 단위  
: 1비트는 2^1로 2개의 상태를 표현하고 2비트는 2^2로 4개의 상태를 표현함  

```
1948년 클로드 섀넌이 정보의 양을 수학적으로 측정하는 방식을 제시하며 정보의 최소 단위로 비트를 정의
```


**Bit Pattern**  
: 여러 개의 비트가 나열되어 특정한 패턴을 이룬 형태  

비트 | 패턴 수 | 거듭 제곱
:---:|:---:|:---:
1  | 2    | 2<sup>1</sup>
2  | 4    | 2<sup>2</sup>
3  | 8    | 2<sup>3</sup>
4  | 16   | 2<sup>4</sup>
8  | 256  | 2<sup>8</sup>
10 | 1024 | 2<sup>10</sup>


**Nibble**  
: 4개 비트 또는 반 바이트로 구성된 단위  


**Byte**  
: 8개 비트로 구성 0000 0000(0) ~ 1111 1111(255)  
: 1바이트는 2^8로 256개의 상태를 표현  
: 1바이트로 영어 알파벳 문자 하나를 표현할 수 있어 1 캐릭터라고 부르기도 함  
: 대부분의 프로그래밍 언어에서 사용하는 데이터 처리 단위  

```
0 0 0 0   0 0 0 0

상위 4비트 = 상위 니블, MSB 
하위 4비트 = 하위 니블, LSB

MSB = 최상위 비트
LSB = 최하위 비트
```


**Word**  
: CPU가 한 번에 처리하는 데이터 크기   
: 시스템 아키텍처에 따라 16비트, 32비트, 64비트 등 표현 가능한 크기가 달라짐  
: 현대 컴퓨터는 64비트 시스템을 기본으로 사용하며 64비트 시스템이 32비트 연산을 지원함  

```
워드 = 16비트
더블 워드 = 32비트
쿼드 워드 = 64비트 
```



#### Logical Unit
: 컴퓨터 시스템에서 데이터의 의미와 구조를 표현하기 위한 추상적 단위 (개념적 요소)  
: 데이터를 사용자 관점에서 의미있게 구조화하여 데이터 처리와 관리를 용이하게 함  
: 물리적 저장소나 저장 구조가 변경되어도 이와 상관없이 데이터 구조를 유지함  

단위 | 설명
:---:|---
field    | 특별한 의미(속성)를 갖는 가장 작은 데이터 단위
record   | 관련된 필드들의 집합
file     | 관련된 레코드들의 집합
database | 관련된 파일들의 집합

**database**  
: 데이터베이스를 이루는 파일들은 데이터 간 논리적 관계가 존재  
: 데이터베이스의 데이터는 중복을 최소화해 무결성과 일관성을 유지해야 함  

**구현**  
: Logical Unit 개념을 기반으로 논리적 구조를 구현해 데이터를 관리하는 시스템

- [Database Management System](./data-storage/dbms/index.md)
