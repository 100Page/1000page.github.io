# Computer Data
: 컴퓨터 시스템에 의해 처리, 저장, 전송되는 모든 형태의 데이터를 의미  
: 컴퓨터는 다양한 종류가 있으며 이에 따라 데이터 표현 방식이나 처리 방식이 달라질 수 있음  

- [Analog Computer](#analog-computer)
- [Digital Computer](#digital-computer)
- [Quantum Computer](#quantum-computer)


**data conversion**
: 데이터를 한 형식에서 다른 형식으로 바꾸는 과정  
: 데이터를 다양한 장치에서 사용하기 위해 형식을 변환  

```
Analog-to-Digital Conversion
: 아날로그 신호를 디지털 신호로 변환하는 과정

Digital-to-Analog Conversion
: 디지털 신호를 아날로그 신호로 변환하는 과정
```


## Analog Computer
: 아날로그 컴퓨터는 온도, 전압, 파장, 소리 등 물리적인 데이터를 처리  
: 입력된 물리적 데이터는 트랜스듀서를 통해 전기적 신호로 변환  
: 이 전기적 신호는 물리적 데이터와 유사한(analog) 연속적인 값  



## Digital Computer
: 디지털 컴퓨터는 입력된 데이터를 디지털 신호로 변환해 처리  
: 디지털 신호란 입력된 데이터를 디지털 데이터로 변환하기 위한 신호로 일반적으로 0과 1을 의미    



### Digital Data
: 이진수 조합을 통해 데이터를 표현하는 것  
: 디지털 컴퓨터는 데이터를 이진법으로 처리하므로 텍스트, 이미지 등 모든 데이터는 이진 형태로 변환되어야 함  
: 전기 회로의 ON/OFF 방식과 이진 데이터의 0과 1이 대응해 전기 신호의 손실이나 적어 보다 안정적임  

```
1. 입력된 데이터를 이진 데이터로 변환해 기억 장치에 저장       
2. 필요에 따라 저장된 데이터를 가공하고
3. 가공된 데이터를 사람이 알아 볼 수 있는 형식으로 변환해 출력
```


**Binary Number**  
: 이진수란 0과 1 두 개의 숫자만을 사용해 수를 표현하는 방식  


**Binary System**  
: 이진법이란 한 자리에 0과 1 숫자만을 사용하고 1을 초과하면 자리 올림하는 표현법  
: 8진법, 10진법 등 다양한 진법 존재하며 다른 진법과의 변환이 용이함  

진법 | 범위 | 표기법(접두사)
:---:|:---:| :---:
2진법  | 01               | 0b
8진법  | 01234567         | 0
10진법 | 0123456789       | -
16진법 | 0123456789abcdef | 0x


진법의 각 숫자는 숫자가 위치한 자리의 값을 갖음
```
123 = 1*10**2 + 2*10**1 + 3*10**0 
1111011 = 1*2**6 + 1*2**5 + 1*2**4 + 1*2**3 + 0*2**2 + 1*2**1 + 1*2**0
```


**Binary Format**  
: 이진 형식이란 데이터를 이진수로 표현하는 형식을 의미  
: 초기 컴퓨터는 천공 카드의 구멍 유무로 이진 형식을 표현했으며 현재는 전류의 유무로 표현  

```
이진수 : 1010
십진수 : 1*2³ + 0*2² + 1*2¹ + 0*2 => 8 + 0 + 2 + 0 = 10
```


**Binary Data**  
: 이진 데이터란 이진 형식으로 변환된 데이터를 의미  
: 가장 기본 단위는 비트로 이진 데이터는 비트 조합으로 구성  

데이터 처리 과정
> 데이터 => 인코딩 => 압축 => 바이너리

```
텍스트 : 안녕

인코딩 : UTF-8 
바이너리 데이터 : 11101100 10000000 10011101 11101101 10000001 10001100

인코딩 : UTF-16
바이너리 데이터 : 11000000 11011101 11011000 01001100
```


**Binary Stream**  
: 이진 데이터의 연속적인 흐름을 의미  
: 파일 입출력, 네트워크 전송 등 데이터 이동이나 처리에 사용  


**Digital Encoding**
: 디지털 부호화란 특정 데이터를 특정한 포맷으로 변환하는 프로세스  
: 데이터를 저장하거나 전송할 때 데이터를 어떻게 이진수로 변환할지 결정

- 문자 인코딩 (ASCII, Unicode)
- 텍스트 인코딩 (Base64, Hex)
- 이미지 인코딩 (jpg, gif)

```
데이터 => 디지털 데이터 => 인코딩 => 디지털 신호
```



#### Digital Data Unit
: 데이터 단위는 물리적 단위와 논리적 단위로 구분  


##### Physical Units
: 물리적 장치에서 사용되는 단위  
: 최소 단위는 비트이며 최근 저장 장치의 용량 증가로 바이트 단위를 기본으로 사용

단위 | 변환 
:---:|:---:
bit      | 1 bit
quarter  | 2 bits
nibble   | 4 bits
Byte     | 8 bits
Word     | 16 bits
Kilobyte | 1024 bytes
Megabyte | 1024 kilobytes
Gigabyte | 1024 megabytes
Terabyte | 1024 gigabytes
Petabyte | 1024 terabytes


**Bit**  
: Binary Digit  
: 0 또는 1의 상태를 갖는 가장 작은 데이터 단위  
: 1비트는 2**1로 2개 데이터를 표현  
: 2비트는 2**2로 4개 데이터를 표현  

```
1948년 클로드 섀넌이 정보의 양을 수학적으로 측정하는 방식을 제시하며 정보의 최소 단위로 비트를 정의
```

**Bit Pattern**  
: 여러 개의 비트가 나열된 상태  

비트 | 패턴 수 | 거듭 제곱
:---:|:---:|:---:
1  | 2    | 2<sup>1</sup>
2  | 4    | 2<sup>2</sup>
3  | 8    | 2<sup>3</sup>
4  | 16   | 2<sup>4</sup>
8  | 256  | 2<sup>8</sup>
10 | 1024 | 2<sup>10</sup>


**Nibble**  
: 4개 비트로 구성 또는 반 바이트로 구성  
: IoT같은 데이터 효율성이 필요한 기기에서 사용  

```
1바이트
- 상위 4비트 = 상위 니블    
- 하위 4비트 = 하위 니블  
```


**Byte**
: 8개 비트로 구성 0000 0000(0) ~ 1111 1111(255)  
: 1바이트는 2**8로 256개의 데이터를 표현   
: 메모리 주소, 대부분의 프로그래밍 언어 등에서 사용하는 가장 작은 데이터 단위  

```
0 0 0 0   0 0 0 0

MSB 최상위 비트 => 가장 왼쪽에 있는 비트
LSB 최하위 비트 => 가장 오른쪽에 있는 비트
```


**Word**  
: CPU가 한 번에 처리하는 데이터 단위  
: 시스템 아키텍처에 따라 16비트, 32비트, 64비트 등 크기가 달라지며 사용 방식이 달라짐  
: 현대 컴퓨터는 64비트 시스템을 기본으로 사용하며 64비트 시스템이 32비트 연산을 지원함  

```
워드 = 16비트
더블 워드 = 32비트
쿼드 워드 = 64비트 
```


###### Endianness
: 엔디안이란 다중 바이트 데이터를 어떻게 저장할지 정의하는 방식  
: 엔디안은 가장 중요한 바이트(MSB)와 가장 덜 중요한 바이트(LSB)의 순서를 결정  
: 시스템 아키텍처에 따라 기본 엔디안 방식이 다름  
: 다른 방식의 엔디안을 사용하는 시스템 간 데이터를 교환하는 경우 바이트 순서를 변환함  

**Big Endian**  
: 가장 중요한 바이트를 먼저 저장  

**Little Endian**  
: 가장 덜 중요한 바이트를 먼저 저장  

```
0x12345678
Big Endian    => 12 34 56 78 
Little Endian => 78 56 34 12 
```

## Quantum Computer
: 양자 컴퓨터는 디지털 컴퓨터와 다른 방식으로 데이터를 처리  
: 양자 컴퓨터는 양자 비트(Qubit)를 통해 데이터를 저장하고 처리  

**양자 비트**  
: 양자 물리학 법칙에 따라 0과 1을 동시에 갖는 중첩 상태가 가능한 데이터 단위
