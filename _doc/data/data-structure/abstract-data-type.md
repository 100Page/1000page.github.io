# Abstract Data Type
: 데이터의 동작, 행위, 규칙을 정의  
: 데이터를 제어하는 기능에 대해 추상적인 정의만 제공함  
: 1960년대 데이터 추상화 개념이 등장했고 1980년대 객체 지향 프로그래밍이 발전하며 함께 발전함  

**구성 요소**
- 데이터 : 데이터와 데이터 상태
- 연산 : 데이터를 조작하는 연산과 연산에 대한 명세

연산 | 설명
---|---
접근 | 자료구조 내 특정 데이터에 접근하는 연산
순회 | 자료구조 내 모든 데이터를 순차적으로 방문하는 연산
검색 | 자료구조에서 특정 데이터를 찾는 연산
정렬 | 자료구조의 데이터를 일정한 순서대로 재배열하는 연산
탐색 | 자료구조 내 모든 데이터를 탐색하며 처리하는 연산
병합 | 여러 데이터를 하나로 합치는 연산
분할 | 데이터를 여러 부분으로 나누는 연산


**특징**
- 추상화 : 데이터 구조의 내부 구현을 숨기고 필수적인 특성만 노출
- 캡슐화 : 데이터와 데이터에 대한 연산을 하나로 묶음
- 유연성 : 내부 구현이 변경되어도 ADT를 사용하는 코드는 영향이 덜 함

**구현체**

ADT | Implementation 
---|---
List  | 배열 기반 리스트, 단순 연결 리스트, 이중 연결 리스트
Stack | 배열 기반 스택, 연결 리스트 기반 스택
Queue | 배열 기반 큐, 연결 리스트 기반 큐, 원형 큐
Priority Queue | 힙, 이진 탐색 트리, 정렬된 배열
Deque | 배열 기반 덱, 이중 연결 리스트 기반 덱
Tree  | 이진 트리, 이진 탐색 트리, 균형 이진 트리, 트라이
Graph | 인접 행렬, 인접 리스트, 간선 리스트
Set   | 해시 집합, 트리 집합
Map   | 해시 맵, 트리 맵, 연결 해시 맵



## List
: 순서가 있는 요소들의 집합  
: 요소의 중복이 가능하며 임의 위치에 접근 가능  

- 삽입
  - 특정 위치에 요소 추가
  - 목록 맨 앞이나 맨 뒤에 요소 추가
- 삭제
  - 특정 위치의 요소 삭제
  - 특정 요소 삭제
  - 모든 요소 삭제
- 조회
  - 특정 위치의 요소 반환
  - 특정 요소의 위치 반환
  - 요소 존재 여부 확인
  - 목록의 사이즈 반환
  - 목록에 요소가 있는지 여부 반환
- 수정
  - 특정 위치의 요소 수정



## Stack
: 순서가 있는 요소들의 집합으로 LIFO 원칙  

**LIFO**  
: Last In First Out  
: 마지막에 삽입된 요소가 가장 먼저 처리됨  

- 삽입
  - 요소 추가
- 삭제
  - 최상단 요소 삭제
  - 모든 요소 삭제
- 조회
  - 최상단 요소 반환
  - 요소 개수 반환
  - 스택에 요소가 있는지 여부 반환



## Queue
: 순서가 있는 요소들의 집합으로 FIFO 원칙  

**FIFO**  
: First In First Out  
: 가장 먼저 삽입된 요소가 가장 먼저 처리됨  

- 삽입
  - 요소 추가
- 삭제
  - 맨 앞 요소 삭제
  - 모든 요소 삭제
- 조회
  - 맨 앞 요소 반환
  - 요소 개수 반환
  - 큐에 요소가 있는지 여부 반환



### Priority Queue
: 순서가 있는 요소들의 집합  
: 각 요소는 우선 순위를 가지며 우선 순위가 높거나 낮은 요소가 먼저 처리되어야 함   

- 삽입
  - 우선 순위를 가진 요소 추가
- 삭제
  - 우선 순위가 가장 높거나 낮은 요소 삭제
  - 큐에 있는 모든 요소 삭제
- 조회
  - 우선 순위가 가장 높거나 낮은 요소 조회
  - 큐에 있는 요소 개수 반환
  - 큐에 요소가 있는지 여부 반환
- 수정
  - 특정 요소의 우선 순위 수정



## Deque 
: 순서가 있는 요소들의 집합으로 양쪽 끝에서 삽입과 삭제가 가능  

- 삽입
  - 요소를 앞이나 뒤에 추가
- 삭제
  - 요소를 앞이나 뒤에서 삭제
  - 덱에 있는 모든 요소 삭제
- 조회
  - 요소를 앞이나 뒤에서 조회
  - 덱에 있는 요소 개수 반환
  - 덱에 요소가 있는지 여부 반환



## Graph
: 요소를 그래프 형태로 표현  

- 삽입
  - 정점 추가, 간선 추가
- 삭제
  - 정점 삭제, 간선 삭제
  - 모든 정점과 간선 삭제
- 조회
  - 모든 정점 반환, 모든 간선 반환
  - 인접 정접 목록 반환
  - 간선 가중치 반환
  - 깊이 우선 탐색
  - 너비 우선 탐색
  - 정점 개수 반환



## Tree
: 요소를 트리 형태로 표현  
: 트리는 그래프의 하위 개념으로 그래프의 특수한 형태를 트리라고 함  

- 삽입
  - 노드 추가
  - 왼쪽 자식 노드 추가
  - 오른쪽 자식 노드 추가
- 삭제
  - 노드 삭제
  - 왼쪽 자식 노드 삭제
  - 오른쪽 자식 노드 삭제
  - 모든 노드 삭제
- 조회
  - 노드 존재 여부 확인
  - 전위/중위/후위 순회
  - 레벨 순회
  - 루트 노드 반환
  - 부모 노드 반환
  - 노드 개수 반환



## Set
: 순서가 없는 요소들의 집합으로 요소의 중복을 허용하지 않음  

- 삽입
  - 요소 추가
- 삭제
  - 요소 삭제
  - 모든 요소 삭제
- 조회
  - 요소 존재 여부 확인
  - 요소 개수 반환
  - 집합에 요소가 있는지 여부 반환
- 연산
  - 합집합, 교집합, 차집합 연산
  - 부분 집합 여부 확인
  - 배열로 변환



## Map
: 요소를 키/값 쌍으로 나타내며 각 키는 유일하며 하나의 값과 매핑됨  

- 삽입
  - 키/값 추가
- 삭제
  - 키/값 삭제
  - 모든 키/값 삭제
- 조회
  - 키에 해당하는 값 반환
  - 키 존재 여부 반환
  - 값 존재 여부 반환
  - 모든 키/값 반환
  - 모든 키 반환
  - 모든 값 반환
  - 키/값 개수 반환
  - 맵에 요소가 있는지 여부 반환
